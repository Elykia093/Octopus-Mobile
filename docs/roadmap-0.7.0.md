# Octopus Mobile 0.7.0 Roadmap

## 目标

0.7.0 聚焦 Site 工作流反馈，把 Web 管理端的签到总览、状态筛选和当前结果可见范围迁到移动端，并为后续同步、导入和账号操作反馈继续铺路。

## 本轮起点

- 0.6.0 已正式发布，Projection/SiteChannel 批量模型操作、账号内过滤、排序和高级设置校验已经落地。
- API 覆盖基线显示 Mobile 覆盖当前 Web `/api/v1` 路径，剩余差距主要是交互行为和运营反馈。
- Web Site 模块已有 `CheckinPanel`，可按今日签到状态查看成功、失败、未执行和禁用账号，并让列表跟随筛选结果收窄。

## 本轮已落地

- 新增移动端 Site 签到状态推导规则，与 Web `checkin-status.ts` 对齐：
  - 不支持自动签到的平台不纳入统计。
  - 禁用站点或账号计入禁用。
  - 当天未执行计入未执行。
  - 当天 `success` 计入成功，`failed` 和 `skipped` 计入失败。
- 新增 Site 签到总览面板，展示当前可见站点/账号、失败数量和状态筛选。
- Site 列表现在会跟随签到状态筛选收窄账号范围，批量站点操作继续跟随当前可见站点范围。
- 新增 Site 签到状态推导、汇总、搜索和筛选单元测试。
- 账号同步结果现在展示服务器消息和分组、Key、模型数量。
- 账号签到结果现在展示状态、服务器消息和奖励；失败结果进入错误提示。

## 后续候选

1. Site 批量同步与批量签到反馈：展示批量任务启动后的重点失败、最近执行消息和恢复入口。
2. Site 手动签到 URL：对齐 Web 的集中入口，但移动端需要确认外部打开方式和安全提示。
3. Site 导入/归档/账号编辑反馈审计：补足成功、失败、空状态和可恢复动作。
